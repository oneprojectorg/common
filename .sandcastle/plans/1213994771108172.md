# Plan: Default Hidden Proposals per Phase

## Problem
Admins in the grantmaking/participatory funding segment need proposals to be hidden by default so they are not publicly visible unless explicitly made visible. Currently, admins must manually hide each proposal after submission, which is cumbersome and leaves a window where proposals are visible.

## Approach
1. **Add `defaultHidden` to PhaseRules.proposals** — a new boolean field in the phase rules that controls whether new proposals in that phase are hidden by default.
2. **Update Zod encoder** — add `defaultHidden: z.boolean().optional()` to `phaseRulesEncoder.proposals` in `services/api/src/encoders/decision.ts`.
3. **Update TypeScript type** — add `defaultHidden?: boolean` to `PhaseRules.proposals` in `packages/common/src/services/decision/schemas/types.ts`.
4. **Add UI toggle** — add a `ToggleButton` labeled "Hide proposals by default" below the "Proposal submission" toggle in `PhaseDetailPage.tsx`. The toggle is only meaningful when proposal submission is enabled.
5. **Apply on submit** — in `submitProposal.ts`, after validation passes, check the current phase's `rules.proposals.defaultHidden`. If true, set `visibility: Visibility.HIDDEN` on the proposal during the submit transaction.
6. **Integration test** — add a test in `submit.test.ts` that enables `defaultHidden` on the phase, submits a proposal, and asserts visibility is HIDDEN.
7. **E2e test** — create a Playwright test that enables the toggle, submits a proposal as a member, and verifies: admin sees it (with hidden label), member who didn't submit doesn't see it, submitter sees their own.

## Files
- `packages/common/src/services/decision/schemas/types.ts` — add `defaultHidden` to `PhaseRules`

- `apps/app/src/components/decisions/ProcessBuilder/stepContent/general/PhaseDetailPage.tsx` — add toggle

- `services/api/src/routers/decision/proposals/submit.test.ts` — integration test
- `tests/e2e/tests/default-hidden-proposals.spec.ts` — e2e test
- `apps/app/src/lib/i18n/dictionaries/*.json` — add translation keys

## Edge cases
- Toggle is off by default (backward compatible — existing processes unchanged)
- If `defaultHidden` is undefined/false, proposals remain VISIBLE (current behavior)
- Admins can still manually unhide individual proposals after submission
- The setting only affects new submissions; existing proposals are not retroactively hidden
- Draft proposals are unaffected (they're already only visible to collaborators)

## Out of scope
- Bulk hide/unhide existing proposals
- Visibility settings at the process level (this is per-phase)
- UI changes to proposal cards (hidden label already exists)
- Changes to proposal creation (only submission triggers the default)
