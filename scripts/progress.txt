## Codebase Patterns
- `SidebarItem.labelKey` is typed as `TranslationKey | string` — use `section.isDynamic ? section.labelKey : t(section.labelKey as TranslationKey)` pattern when rendering
- `useProcessNavigation` accepts optional `phases: PhaseNavItem[]` param; dynamic phase sections are inserted between 'phases' and 'proposalCategories' in the navigation order
- Phase section IDs follow pattern `phase-{phaseId}` — use `phaseToSectionId()` / `sectionIdToPhaseId()` / `isPhaseSection()` helpers from navigationConfig.ts
- Sidebar phase data uses 3-tier priority: Zustand store > instanceData.phases > processSchema.phases
- `getContentComponentFlat` checks `isPhaseSection()` first for dynamic phase sections, then falls back to flat registry
- Phase detail saves the full phases array to the API (not just the single phase) — the API expects all phases in the update
- `tsgo` caches JSON imports in `tsconfig.tsbuildinfo` — delete that file if new dictionary keys aren't recognized by typecheck
